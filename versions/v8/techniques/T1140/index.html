Deobfuscate/Decode Files or Information

Adversaries may use Obfuscated Files or Information to hide artifacts of an intrusion from analysis. They may require separate mechanisms to decode or deobfuscate that information depending on how they intend to use it. Methods for doing that include built-in functionality of malware or by using utilities present on the system.

One such example is use of certutil to decode a remote access tool portable executable file that has been hidden inside a certificate file. [1] Another example is using the Windows copy /b command to reassemble binary fragments into a malicious payload. [2]

Sometimes a user's action may be required to open it for deobfuscation or decryption as part of User Execution. The user may also be required to input a password to open a password protected compressed/encrypted file that was provided by the adversary. [3]

Rising Sun decrypted itself using a single-byte XOR scheme. Additionally, Rising Sun can decrypt its configuration data at runtime.[91]

Rocke

Rocke has extracted tar.gz files after downloading them from a C2 server.[92]
RogueRobin

RogueRobin decodes an embedded executable using base64 and decompresses it.[93]
Sandworm Team

Sandworm Team's VBS backdoor can decode Base64-encoded data and save it to the %TEMP% folder. The group also decrypted received information using the Triple DES algorithm and decompresses it using GZip.[94][95]
SDBot

SDBot has the ability to decrypt and decompress its payload to enable code execution.[96][97]
Shamoon

Shamoon decrypts ciphertext using an XOR cipher and a base64-encoded string.[98]

ShimRat

ShimRat has decompressed its core DLL using shellcode once an impersonated antivirus component was running on a system.[99]
Skidmap

Skidmap has the ability to download, unpack, and decrypt tar.gz files .[100]

Smoke Loader

Smoke Loader deobfuscates its code.[101]
SoreFang

SoreFang can decode and decrypt exfiltrated data sent to C2.[102]
SQLRat

SQLRat has scripts that are responsible for deobfuscating additional scripts.[103]
Starloader

Starloader decrypts and executes shellcode from a file called Stars.jps.[104]
Sunspot

Sunspot decrypts Sunburst, which was stored in AES128-CBC encrypted blobs.[105]

Teardrop

Teardrop was decoded using a custom rolling XOR algorithm to execute a customized Cobalt Strike payload.[106][107][80]
Threat Group-3390

During execution, Threat Group-3390 malware deobfuscates and decompresses code that was encoded with Metasploit’s shikata_ga_nai encoder as well as compressed with LZNT1 compression.[108]
TrickBot

TrickBot decodes the configuration data and modules.[109][110]
Tropic Trooper

Tropic Trooper used shellcode with an XOR algorithm to decrypt a payload. Tropic Trooper also decrypted image files which contained a payload.[111][112]
TSCookie

TSCookie has the ability to decrypt, load, and execute a DLL and its resources.[113]
Turla

Turla has used a custom decryption routine, which pulls key and salt values from other artifacts such as a WMI filter or PowerShell Profile, to decode encrypted PowerShell payloads.[114]
TYPEFRAME

One TYPEFRAME variant decrypts an archive using an RC4 key, then decompresses and installs the decrypted malicious DLL module. Another variant decodes the embedded file by XORing it with the value "0x35".[115]
UNC2452

UNC2452 used 7-Zip to decode its Raindrop malware.[79]
Ursnif

Ursnif has used crypto key information stored in the Registry to decrypt Tor clients dropped to disk.[116]
Valak

Valak has the ability to decode and decrypt downloaded files.[117][118]
VERMIN

VERMIN decrypts code, strings, and commands to use once it's on the victim's machine.[119]
Volgmer

Volgmer deobfuscates its strings and APIs once its executed.[120]
WellMail

WellMail can decompress scripts received from C2.[121]
WellMess

WellMess can decode and decrypt data received from C2.[122][123][124]
WindTail

WindTail has the ability to decrypt strings using hard-coded AES keys.[125]
Winnti for Linux

Winnti for Linux has decoded XOR encoded strings holding its configuration upon execution.[126]
WIRTE

WIRTE has decoded a base64 encoded document which was embedded in a VBS script.[127]
YAHOYAH

YAHOYAH decrypts downloaded files before execution.[128]
Zebrocy

Zebrocy decodes its secondary payload and writes it to the victim’s machine. Zebrocy also uses AES and XOR to decrypt strings and payloads.[129][130]
ZeroT

ZeroT shellcode decrypts and decompresses its RC4-encrypted payload.[131]
Zeus Panda

Zeus Panda decrypts strings in the code during the execution process.[132]

Mitigations

This type of attack technique cannot be easily mitigated with preventive controls since it is based on the abuse of system features.

Detection

Detecting the action of deobfuscating or decoding files or information may be difficult depending on the implementation. If the functionality is contained within malware and uses the Windows API, then attempting to detect malicious behavior before or after the action may yield better results than attempting to perform analysis on loaded libraries or API calls. If scripts are used, then collecting the scripts for analysis may be necessary. Perform process and command-line monitoring to detect potentially malicious behavior related to scripts and system utilities such as certutil.

Monitor the execution file paths and command-line arguments for common archive file applications and extensions, such as those for Zip and RAR archive tools, and correlate with other suspicious behavior to reduce false positives from normal user and administrator behavior.
