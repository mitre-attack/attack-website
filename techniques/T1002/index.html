Data Compressed

An adversary may compress data (e.g., sensitive documents) that is collected prior to exfiltration in order to make it portable and minimize the amount of data sent over the network. The compression is done separately from the exfiltration channel and is performed using a custom program or algorithm, or a more common compression library or utility such as 7zip, RAR, ZIP, or zlib.

ID: T1002

Tactic: Exfiltration

Platform:  Linux, Windows, macOS

Data Sources:  Binary file metadata, File monitoring, Process command-line parameters, Process monitoring

Requires Network:  No

Version: 1.0

Calisto uses the zip -r command to compress the data collected on the local system.[10][11]

Cardinal RAT applies compression to C2 traffic using the ZLIB library.[12]

CopyKittens uses ZPP, a .NET console program, to compress files with ZIP.

CORALDECK has created password-protected RAR, WinImage, and zip archives to be exfiltrated.[13]

Denis compressed collected data using zlib.[15]

Dragonfly 2.0

Dragonfly 2.0 compressed data into .zip files prior to exfiltrating it.[16]

Duqu

Modules can be pushed to and executed by Duqu that copy data to a staging area, compress it, and XOR encrypt it.[17]

Empire

Empire can ZIP directories on the target system.[18]

Epic

Epic compresses the collected data with bzip2 before sending it to the C2 server.[19]

FIN6

Following data collection, FIN6 has compressed log files into a ZIP archive prior to staging and exfiltration.[20]

FIN8

FIN8 has used RAR to compress collected data before.[21]

Gallmaker

Gallmaker has used WinZip, likely to archive data prior to exfiltration.[22]

Honeybee

Honeybee adds collected files to a temp.zip file saved in the %temp% folder, then base64 encodes it and uploads it to control server.[23]

iKitten

iKitten will zip up the /Library/Keychains directory before exfiltrating it.[24]

InvisiMole

InvisiMole uses WinRAR to compress data that is intended to be exfiltrated.[25]

Ke3chang

The Ke3chang group has been known to compress data before exfiltration.[26]

Lazarus Group

Lazarus Group malware IndiaIndia saves information gathered about the victim to a file that is compressed with Zlib, encrypted, and uploaded to a C2 server. Lazarus Group malware RomeoDelta archives specified directories in .zip format, encrypts the .zip file, and uploads it to its C2 server.[27][28][29]

Lurid

Lurid can compress data before sending it.[30]

Magic Hound

Magic Hound has used RAR to stage and compress local folders.[31]

menuPass

menuPass has compressed files before exfiltration using TAR and RAR.[32][33]

Micropsia

Micropsia creates a RAR archive based on collected files on the victim's machine.[34]

MuddyWater

MuddyWater has used the native Windows cabinet creation tool, makecab.exe, likely to compress stolen data to be uploaded.[35]

OopsIE

OopsIE compresses collected files with both the GZipStream class and a simple character replacement scheme before sending them to its C2 server.[36]

PoshC2

PoshC2 contains a module for compressing data using ZIP.[37]

Prikormka

After collecting documents from removable media, Prikormka compresses the collected files.[38]

Proton

Proton zips up files before exfiltrating them.[24]

Pupy

Pupy can compress data with Zip before sending it over C2.[39]

RunningRAT

RunningRAT contains code to compress files.[40]

SeaDuke

SeaDuke compressed data with zlib prior to sending it over C2.[41]

Sowbug

Sowbug extracted documents and bundled them into a RAR archive.[42]

Threat Group-3390

Threat Group-3390 has used RAR to compress, encrypt, and password-protect files prior to exfiltration.[43]

ZLib

The ZLib backdoor compresses communications using the standard Zlib compression library.[44]

Mitigation

Identify unnecessary system utilities, third-party tools, or potentially malicious software that may be used to compress files, and audit and/or block them by using whitelisting [45] tools, like AppLocker, [46] [47] or Software Restriction Policies [48] where appropriate. [49]

If network intrusion prevention or data loss prevention tools are set to block specific file types from leaving the network over unencrypted channels, then an adversary may move to an encrypted channel.

Detection

Compression software and compressed files can be detected in many ways. Common utilities that may be present on the system or brought in by an adversary may be detectable through process monitoring and monitoring for command-line arguments for known compression utilities. This may yield a significant amount of benign events, depending on how systems in the environment are typically used.

If the communications channel is unencrypted, compressed files can be detected in transit during exfiltration with a network intrusion detection or data loss prevention system analyzing file headers. [50]
