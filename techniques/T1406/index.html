Obfuscated Files or Information

Sub-techniques (2)
ID Name
T1406.001 Steganography
T1406.002 Software Packing

Adversaries may attempt to make a payload or file difficult to discover or analyze by encrypting, encoding, or otherwise obfuscating its contents on the device or in transit. This is common behavior that can be used across different platforms and the network to evade defenses.

Payloads may be compressed, archived, or encrypted in order to avoid detection. These payloads may be used during Initial Access or later to mitigate detection. Portions of files can also be encoded to hide the plaintext strings that would otherwise help defenders with discovery. Payloads may also be split into separate, seemingly benign files that only reveal malicious functionality when reassembled.[1]

S1061 AbstractEmu

AbstractEmu has encoded files, such as exploit binaries, to potentially use during and after the rooting process.[2]
S1095 AhRat

AhRat can use an encryption key received from its C2 to encrypt and decrypt configuration files and exfiltrated data.[3]
S0525 Android/AdDisplay.Ashas

Android/AdDisplay.Ashas has hidden the C2 server address using base-64 encoding. [4]
S1214 Android/SpyAgent

Android/SpyAgent has used the Tencent packer to hide its malicious payload.[5]

S0524 AndroidOS/MalLocker.B

AndroidOS/MalLocker.B has employed both name mangling and meaningless variable names in source. AndroidOS/MalLocker.B has stored encrypted payload code in the Assets directory, coupled with a custom decryption routine that assembles a .dex file by passing data through Android Intent objects. [1]
S0540 Asacub

Asacub has stored encrypted strings in the APK file.[6]
S0293 BrainTest

BrainTest stores a secondary Android app package (APK) in its assets directory in encrypted form, and decrypts the payload at runtime.[7]
S1094 BRATA

BRATA has employed code obfuscation and encryption of configuration files.[8][9]
S0432 Bread

Bread uses various tricks to obfuscate its strings including standard and custom encryption, programmatically building strings at runtime, and splitting unencrypted strings with repeated delimiters to break up keywords. Bread has also abused Java and JavaScript features to obfuscate code. Bread payloads have hidden code in native libraries and encrypted JAR files in the data section of an ELF file. Bread has stored DEX payloads as base64-encoded strings in the Android manifest and internal Java classes.[10][11]
C0033 C0033

During C0033, PROMETHIUM used StrongPity to obfuscate code and strings to evade detection.[12]

S0529 CarbonSteal

CarbonSteal has used incorrect file extensions and encryption to hide most of its assets, including secondary APKs, configuration files, and JAR or DEX files.[13]
S0480 Cerberus

Cerberus uses standard payload and string obfuscation techniques.[14]
S0323 Charger

Charger encodes strings into binary arrays to make it difficult to inspect them. It also loads code from encrypted resources dynamically and includes meaningless commands that mask the actual commands passing through.[15]
S0555 CHEMISTGAMES

CHEMISTGAMES has encrypted its DEX payload.[16]
S0550 DoubleAgent

DoubleAgent has used an AES encrypted file in the assets folder with an unsuspecting name (e.g. ‘GoogleMusic.png’) for holding configuration and C2 information.[13]
S1054 Drinik

Drinik has used custom encryption to hide strings, potentially to evade antivirus products.[17]
S0420 Dvmap

Dvmap decrypts executables from archive files stored in the assets directory of the installation binary.[18]
S0478 EventBot

EventBot dynamically loads its malicious functionality at runtime from an RC4-encrypted TTF file. EventBot also utilizes ProGuard to obfuscate the generated APK file.[19]
S0509 FakeSpy

FakeSpy stores its malicious code in encrypted asset files that are decrypted at runtime. Newer versions of FakeSpy encrypt the C2 address.[20]
S0408 FlexiSpy

FlexiSpy encrypts its configuration file using AES.[21]
S1067 FluBot

FluBot can obfuscated class, string, and method names in newer malware versions.[22]
S0423 Ginp

Ginp obfuscates its payload, code, and strings.[23]
S0421 GolfSpy

GolfSpy encodes its configurations using a customized algorithm.[24]
S0536 GPlayed

GPlayed has base64-encoded the exfiltrated data, replacing some of the base64 characters to further obfuscate the data.[25]
S0406 Gustuff

Gustuff obfuscated command information using a custom base85-based encoding.[26]
S0544 HenBox

HenBox has obfuscated components using XOR, ZIP with a single-byte key or ZIP/Zlib compression wrapped with RC4 encryption.[27]
S0463 INSOMNIA

INSOMNIA obfuscates various pieces of information within the application.[28]

S1185 LightSpy

Using an XOR-chain algorithm, LightSpy decrypts an embedded configuration blob containing URLs for jailbreak components and next-stage payloads. It also decrypts modules in memory and on disk using AES-ECB with the hardcoded key 3e2717e8b3873b29.[29][30][31][32] Additionally, LightSpy’s plugins have been encrypted during transmission.[32]
S0485 Mandrake

Mandrake obfuscates its hardcoded C2 URLs.[33]
S0407 Monokle

Monokle uses XOR to obfuscate its second stage binary.[34]
S0286 OBAD

OBAD contains encrypted code along with an obfuscated decryption routine to make it difficult to analyze.[35]
S0399 Pallas

Pallas stores domain information and URL paths as hardcoded AES-encrypted, base64-encoded strings.[36]
S0539 Red Alert 2.0

Red Alert 2.0 has stored data embedded in the strings.xml resource file.[37]
S0411 Rotexy

Starting in 2017, the Rotexy DEX file was packed with garbage strings and/or operations.[38]
S1055 SharkBot

SharkBot can use a Domain Generation Algorithm to decode the C2 server location.[39]

S0549 SilkBean

SilkBean has hidden malicious functionality in a second stage file and has encrypted C2 server information.[13]
S1195 SpyC23

SpyC23 has used obfuscation techniques to hide its hardcoded C2 address.[40]
S0545 TERRACOTTA

TERRACOTTA has stored encoded strings.[41]
S1056 TianySpy

TianySpy has encrypted C2 details, email addresses, and passwords.[42]

S0427 TrickMo

TrickMo contains obfuscated function, class, and variable names, and encrypts its shared preferences using Java’s PBEWithMD5AndDES algorithm.[43]
G0112 Windshift

Windshift has encrypted application strings using AES in ECB mode and Blowfish, and stored strings encoded in hex during Operation BULL. Further, in Operation BULL, encryption keys were stored within the application’s launcher icon file.[44]
S0312 WireLurker

WireLurker obfuscates its payload through complex code structure, multiple component versions, file hiding, code obfuscation and customized encryption to thwart anti-reversing.[45]
S0489 WolfRAT

WolfRAT’s code is obfuscated.[46]
S0318 XLoader for Android

XLoader for Android loads an encrypted DEX code payload.[47]
S0494 Zen

Zen base64 encodes one of the strings it searches for.[48]

Mitigations

This type of attack technique cannot be easily mitigated with preventive controls since it is based on the abuse of system features.

Detection

ID Data Source Data Component Detects
DS0041 Application Vetting API Calls

Dynamic analysis, when used in application vetting, may in some cases be able to identify malicious code in obfuscated or encrypted form by detecting the code at execution time (after it is deobfuscated or decrypted). Some application vetting techniques apply reputation analysis of the application developer and can alert to potentially suspicious applications without actual examination of application code.
